## üìù Contributing to Documentation

When adding new documentation:

1. **Place it in `yt_serve/DOCS/`** - Keep all docs organized
2. **Use descriptive filenames** - Make it easy to find
3. **Update this index** - Add your doc to the appropriate section
4. **Follow the style** - Match existing documentation format
5. **Link related docs** - Help users navigate

### Documentation Style Guide

- Use clear, concise language
- Include code examples where helpful
- Add screenshots for UI features
- Provide step-by-step instructions
- Link to related documentation
- Keep it up-to-date
